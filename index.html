Was ist Git ?

Git und Github sind nicht das gleiche! Git ist ein VSC-Tool und GitHub ist eine Plattform die Git benutzt, es gibt auch andere zb. GitLab, GitBucket, BitBucket, Sourceforge und so weiter. Git wird überall benutzt und ist eins der wichtigsten Tools für Entwickler geworden. Dadurch können Entwickler von überall zusammenarbeiten. Git ist ein Versionverwaltungstool, das bei jedem comitten eine Art Schnappschuss von deinem Projekt macht. Hinterher hat man einen Stapel von Schnappschüssen.

Git-Befehle

git init

Initialisiert ein lokales git repository

git add .

Gibt an was alles in den commit rein soll

git commit -m “message”

Hinterlegt einen message/commit

git push

Schickt das lokale repository in das github repository

git log

Zeigt alle die bis dato erstellten commits ausführlich an

git log --oneline

Zeigt alle die bis dato erstellten commits in kurzform an

git branch “name”

Erstellt einen neuen Branch

git branch -a

Zeigt alle erstellten Branches an

git branch -d “name”

Löscht den ausgewählten Branch

git checkout

Wechseln zwischen den Branches

git pull

Lädt die aktuelle Version des Repository von GitHub runter

git clone

Kopiert ein fremdes Projekt von Github

git merge

Ist dafür da um zwei branches zusammenzuführen

Abkürzungen

A - Added (This is a new file that has been added to the repository)

M - Modified (An existing file has been changed)

D - Deleted (a file has been deleted)

Untracked (The file is new or has been changed but has not been added to the repository yet)

C - Conflict (There is a conflict in the file)

R - Renamed (The file has been renamed)

S - Submodule (In repository exists another subrepository)

Was ist Terminal ?

Ein Terminal, auch als Konsole eines Computers oder traditionell im deutschen Sprachgebrauch als Datensichtgerät bezeichnet, ist ein Endgerät zur Eingabe und Anzeige von Daten. Terminals wurden entwickelt, um die vormals an Großrechnern eingesetzten Fernschreiber zu ersetzen. Die Abkürzung tty des englischen Worts für Fernschreiber „teletype“ ist noch heute in Unix-Betriebssystemen und entsprechender Software allgegenwärtig. Während in der Frühzeit noch den Fernschreibern vergleichbare Geräte – bezeichnet als „Hardcopy-Terminals“, bestehend aus einem Drucker und einer Tastatur – in Gebrauch waren, bestand der Vorteil von Terminals später darin, dass ein Bildschirm für die Wiedergabe der Daten eingesetzt wurde. Bessere Geräte konnten und können nicht nur sequentiell Zeilen ausgeben: Mit einem beliebig manövrierbaren Cursor konnte Text in jeder Zeile eingegeben und verändert werden. Die ab etwa 1977 allmählich einsetzende Verbreitung der Personal Computer hat die Terminals mittlerweile aus den meisten Büros verdrängt. Viele Terminals sind durch Terminalemulationen ersetzt worden, die auf einem PC laufen. Bis heute sind Terminals aber in einigen Bereichen verbreitet.

Terminal-Befehle

touch

Erstellt eine Datei!

mkdir 'make directory'

Erstellt einen neuen Ordner

ls 'list directory'

Zeigt uns alle Dateien/Ordner im Verzeichnis an

ls -l 'list directory'

Zeigt uns alle Dateien/Ordner im Verzeichnis an, nur ausführlicher

ls -a 'list directory'

Zeigt uns alle Dateien/Ordner, auch die verstecken

cd 'change directory'

Mit cd können in unserem Verzeichnis navigieren

cat 'concatenate'

Den Inhalt von Datein im Terminal anzeigen lassen

mv 'move/rename'

Verschiebt Datein und kann sie umbenennen

cp 'copy files and directories'

Erstellt eine Kopie von unserer Datei oder Ordner

pwd 'print working directory'

Zeigt uns vom root verzeichnis aus wo wir uns gerade befinden
